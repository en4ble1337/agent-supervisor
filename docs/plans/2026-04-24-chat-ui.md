# Implementation Plan: Frontend Chat Interface

## 1. Objective
Build the direct 1-on-1 chat interface to communicate with agents (`POST /api/agents/{id}/chat`, `GET /api/agents/{id}/chat`).

## 2. Proposed Changes
- `frontend/src/api.ts`: Add `getChatHistory(id)` and `sendMessage(id, content)` functions.
- `frontend/src/components/ChatTerminal.tsx`: Main chat interface component.
- `frontend/src/pages/AgentDetail.tsx`: Render `ChatTerminal` in the Chat tab.
- `frontend/src/tests/ChatTerminal.test.tsx`: Tests for sending messages and history display.

## 3. Step-by-Step Execution Plan

### Step 1: API and Component
- **Task 1: [RED]** Write test for `api.ts` `getChatHistory` and `sendMessage`.
- **Task 2: [GREEN]** Implement `getChatHistory` and `sendMessage` in `api.ts`.
- **Task 3: [RED]** Write test `ChatTerminal.test.tsx` verifying history loading and message sending.
- **Task 4: [GREEN]** Implement `ChatTerminal.tsx`. Use a scrollable message list and an input field at the bottom. Style messages differently based on role (User vs Agent).
- **Task 5: [REFACTOR]** Ensure auto-scroll to bottom when new messages arrive.

### Step 2: Integration
- **Task 6: [REFACTOR]** Update `AgentDetail.tsx` to render `ChatTerminal` in Chat tab.

## 4. Verification Strategy
- **Unit Tests:** `npm run test` with Vitest.
- **Type Checking:** `npx tsc --noEmit`.

## 5. Potential Risks & Mitigations
- **Risk:** Chat history fetching every time the tab is switched might be redundant.
- **Mitigation:** For MVP, fetching on mount of `ChatTerminal` is fine. If needed, we can lift state to `AgentDetail` later.
